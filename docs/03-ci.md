# 3. CI

**Status :** ⏳ en cours — workflow créé, à valider sur push/PR

---

## But

Pipeline unique qui valide le code (lint, tests unitaires + intégration), build l’app et l’image Docker, puis vérifie que le conteneur répond (smoke). Garantit que chaque push/PR est déployable.

---

## Concepts clés

- **Lint** : `npm run lint` (Next.js ESLint).
- **Tests** : unit (Jest/Vitest) + intégration si présents ; à ajouter si absents.
- **Build** : `npm run build` (mode standalone, sans GITHUB_PAGES).
- **Docker build** : `docker build` de l’image, optionnellement tag pour registry.
- **Smoke** : lancer le conteneur, curl/get sur http://localhost:3000, vérifier 200.

---

## Fichiers créés

| Fichier | Rôle |
|---------|------|
| `.github/workflows/ci.yml` | 3 jobs : **lint_test** (npm ci, lint, tests si script présent) → **build** (npm run build) → **docker_smoke** (docker build + curl :3000). Job summary par job. |

---

## Structure du workflow

| Job | Étapes |
|-----|--------|
| **lint_test** | Checkout → Setup Node → `npm ci` → détection script `test` → `npm run lint` → `npm test` (si script présent, sinon skip sans fail) → Job summary |
| **build** | (needs: lint_test) Checkout → Setup Node → `npm ci` → `npm run build` (DOCKER_BUILD=1) → Job summary |
| **docker_smoke** | (needs: build) Checkout → `docker build -t portfolio:ci .` → run container → curl 200 sur :3000 → stop/rm → Job summary |

Chaque job écrit un **Job summary** (tableau des étapes OK / ❌) dans `$GITHUB_STEP_SUMMARY`.

## Troubleshooting

| Problème | Piste |
|----------|--------|
| Workflow ne se déclenche pas | Vérifier `on: push/pull_request` et la branche. |
| Docker build échoue (workerd) | Même correctif que [02-docker.md](02-docker.md) : DOCKER_BUILD=1. |
| Smoke timeout | Augmenter le délai avant curl ou utiliser `wait-on` ; vérifier que le port exposé est 3000. |
