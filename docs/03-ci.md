## But

Valider le code (lint + tests) et build l’app. Le build d’image Docker et le smoke test sont dans le workflow Security.

---

## Concepts clés

- **Lint** : `npm run lint` (Next.js ESLint), job dédié.
- **Tests** : Jest + React Testing Library ; `npm test` (local), `npm run test:ci` (CI + coverage).
- **Build** : `npm run build` (mode standalone), après lint + tests.
- **Smoke** : dans workflow Security (build image + scans + smoke).

## Structure du workflow

| Job | Étapes |
|-----|--------|
| **lint** | Checkout → Setup Node → `npm ci` → `npm run lint` → Job summary |
| **tests** | Checkout → Setup Node → `npm ci` → `npm run test:ci` → Job summary |
| **build** | (needs: lint, tests) Checkout → Setup Node → `npm ci` → `npm run build` (DOCKER_BUILD=1) → Job summary |

## Troubleshooting

| Problème | Piste |
|----------|--------|
| Workflow ne se déclenche pas | Vérifier `on: push/pull_request` et la branche `main`. |
| Tests timeout / open handles | `test:ci` utilise `--forceExit` ; en local, vérifier mocks (next-themes, etc.). |
| Docker build échoue (workerd) | Voir [02-docker.md](02-docker.md) : DOCKER_BUILD=1. |
| Smoke timeout | Délai 10 s avant curl ; augmenter si besoin. |
